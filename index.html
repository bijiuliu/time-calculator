      时间计算器说明

      时间计算器用于处理日常时间换算、时间间隔、前移/后退推算和计算记录保存，适合工作、生活、值班、设备运行、提醒安排等场景。

      一、时间差计算

      用于计算两个时间点之间相差多久。

      例：08:30 到 17:45 = 9小时15分。
      例：22:00 到 06:30，系统会自动按跨天计算，结果为 8小时30分。

      计算结果支持点击切换显示方式，可在“小时+分钟”和“总分钟”之间切换。

      二、前移 / 后退计算

      用于根据一个基准时间，向前或向后推算目标时间。

      前移：08:30 前移 1小时20分 = 07:10。
      后退：08:30 后退 1小时20分 = 09:50。
      跨天：00:30 前移 1小时 = 23:30。

      适合倒推开始时间、安排提醒时间、推算结束时间、巡检节点或检修节点。

      三、连续累计计算

      前移 / 后退模式支持连续累计。

      开启时：08:00 后退15分钟 = 08:15，结果会自动回填为新的基准时间，再点一次 = 08:30。
      关闭时：计算后只显示结果，不会自动修改基准时间。

      可在右上角“⚙ 设置”中开启或关闭连续累计。

      四、时间输入方式

      支持数字输入和系统选择器两种输入方式。

      数字输入：手动输入小时和分钟，适合快速键入。
      系统选择器：调用浏览器或手机自带时间选择器，适合点选时间。

      日期计算器说明

      日期计算器用于处理日期间隔、日期前移/后退、周期推算和日期记录保存，适合计划安排、假期计算、项目周期、到期日和提醒日推算。

      一、日期差计算

      用于计算两个日期之间相差多少天。

      例：2026年1月1日 到 2026年1月31日 = 相差30天。
      例：2026年6月1日 到 2026年6月10日 = 相差9天。

      适合计算距离某天还有多久、项目周期、假期天数或两个日期之间的间隔。

      二、日期前移 / 后退

      用于根据一个基准日期，向前或向后推算目标日期。

      前移：2026年6月10日 前移7天 = 2026年6月3日。
      后退：2026年6月10日 后退30天 = 2026年7月10日。
      跨年：2026年1月1日 前移1天 = 2025年12月31日。

      适合推算到期日、提醒日、复查日、计划开始日或计划结束日。

      三、连续累计计算

      日期前移 / 后退模式支持连续累计。

      开启时：2026年6月10日 后退7天 = 2026年6月17日，结果会自动回填为新的基准日期，再点一次 = 2026年6月24日。
      关闭时：计算后只显示结果，不会自动修改基准日期。

      可在右上角“⚙ 设置”中开启或关闭连续累计。

      四、日期输入方式

      支持数字输入和系统选择器两种日期输入方式。

      数字输入：分别输入年、月、日，适合精确输入。
      系统选择器：调用浏览器或手机自带日期选择器，适合快速点选日期。

      切换输入方式时，已填写的有效日期会尽量同步到另一种输入方式。

      设置功能说明

      右上角“⚙ 设置”中可以调整工具的输入、外观、计算、交互和记录相关选项。设置采用二级菜单结构，进入对应分类后修改，最后点击“应用”生效。

      一、输入设置

      可设置输入方式、默认打开页面和默认计算模式。

      默认打开可选择时间计算器、日期计算器，或记住上次使用页面。
      默认模式可选择时间差/日期差、前移/后退，或记住上次模式。

      二、外观设置

      支持浅色、深色、跟随系统三种显示模式，也支持标准和紧凑两种界面密度。

      三、计算设置

      可设置前移/后退是否连续累计，也可以设置计算记录保留条数。

      记录条数支持 10 条、20 条或 30 条。

      四、交互设置

      可设置应用后的行为、按钮震动反馈和更新日志弹窗。

      应用后退出设置界面：点击应用后直接返回主界面。
      应用后不退出设置界面：点击应用后保留在当前二级菜单页。

      五、恢复默认与应用

      点击“恢复默认”只会先把设置项填成默认选项，需要再点击“应用”后才会真正保存并生效。

      记录与本地保存

      计算记录会保存在当前浏览器本地，时间记录和日期记录分开保存，互不影响。

      一、记录保存

      每次成功计算后，会自动保存一条对应记录。

      时间计算结果会进入时间记录，日期计算结果会进入日期记录。

      二、记录操作

      记录支持复制、删除和一键清空。

      清空记录前会弹出确认窗口，避免误删。

      三、记录条数

      可在“设置 → 计算设置”中选择记录保留条数。

      支持 10 条、20 条或 30 条。

      四、本地保存说明

      设置、计算记录和部分页面状态保存在当前浏览器本地。更换浏览器、清除浏览器数据、使用无痕模式或更换打开环境后，数据可能会丢失或无法同步。

      注意事项

      一、时间跨天规则

      时间差计算中，如果结束时间小于开始时间，系统会自动认为是跨天计算。

      例：22:00 到 06:30，会按第二天 06:30 计算。

      二、日期有效性

      请输入有效日期。例如闰年 2024年2月29日 是有效日期，但 2026年2月29日 不是有效日期。

      三、震动反馈

      按钮震动反馈仅在支持震动 API 的设备和浏览器中生效，iPhone 通常不支持网页震动。

      四、正式用途提醒

      涉及考勤、工资、生产记录、设备运行记录、合同期限、项目周期等正式用途时，请以实际记录、公司制度、合同条款或官方系统数据为准。
